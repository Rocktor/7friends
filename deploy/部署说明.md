# 部署说明（Ubuntu + nginx + systemd）

## 0. 要求

- **Node ≥ 22**（用到内置 `node:sqlite`）。检查：`node -v`。
- nginx（做反向代理 + WebSocket 升级）。

## 1. 拉代码并打包

```bash
git clone https://github.com/Rocktor/7friends.git
cd 7friends/server
npm ci            # 安装依赖（esbuild / ws / typescript）
npm run build     # 生成 dist/server.mjs（已内联 engine 与 ws，单文件可跑）
```

打包产物 `server/dist/server.mjs` 自带依赖，运行时只需 Node（无需 node_modules）。

## 2. 跑一下确认

```bash
cd 7friends/server
CLIENT_HTML=$(pwd)/../client/index.html PORT=5080 DB_PATH=$(pwd)/zpy.db \
  node --experimental-sqlite dist/server.mjs
# 看到 “[zpy] 7friends server on :5080 …” 即正常
```

环境变量：

| 变量 | 默认 | 说明 |
|------|------|------|
| `PORT` | 5080 | 监听端口（建议只在内网，由 nginx 转发） |
| `DB_PATH` | `./zpy.db` | SQLite 文件路径 |
| `CLIENT_HTML` | `../client/index.html` | 客户端页面路径 |

## 3. systemd 守护

把 `deploy/zpy-server.service` 改好路径后放到 `/etc/systemd/system/`：

```bash
sudo cp deploy/zpy-server.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now zpy-server
sudo systemctl status zpy-server
```

## 4. nginx 反向代理（含 WebSocket 升级）

参考 `deploy/nginx.conf.example`，关键是把 `/ws` 的 `Upgrade` / `Connection` 头透传，否则连不上对局。

```bash
sudo cp deploy/nginx.conf.example /etc/nginx/sites-available/7friends
# 改好 server_name 与 proxy_pass 端口后：
sudo ln -s /etc/nginx/sites-available/7friends /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
```

## 5. 更新

```bash
cd 7friends && git pull
cd server && npm ci && npm run build
sudo systemctl restart zpy-server
```
